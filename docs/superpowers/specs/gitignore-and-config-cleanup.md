# Spec: Gitignore & Config Cleanup for `spire init`

**Date:** 2026-03-19
**Status:** Ready for implementation

## Problem

Several machine-specific files are committed to git that break on fresh clones or forks:

- `.mcp.json` — hardcoded absolute path (`/Users/jb/awell/spire/packages/mcp-server/index.js`)
- `.cursor/mcp.json` — same hardcoded path
- `.envrc` — contains `SPIRE_IDENTITY="spi"`, only correct for the hub
- `.claude/settings.local.json` — local editor config with machine-specific MCP server list
- `.beads/metadata.json` — instance-specific `project_id` UUID and `dolt_database` name
- `.beads/routes.jsonl` — hub topology (3 prefixes), not valid for fresh clones

All of these are already generated by `spire init`. They shouldn't be in version control.

Additionally, `.beads/.beads-credential-key` (a raw encryption key) is untracked but not
explicitly gitignored — a `git add .` would commit it.

## Design principle

**Git tracks schema. `spire init` generates instance state.**

After this change, `git clone` gives you everything you need to *run* `spire init`.
`spire init` generates everything you need to *use* spire.

What git tracks in `.beads/`:
- `.gitignore` (beads manages its own)
- `README.md`
- `config.yaml` (repo-level defaults — routing mode, backup settings, etc.)
- `hooks/` (git hooks)
- `formulas/` (workflow molecules)

What git tracks in the repo root (managed by spire):
- `SPIRE.md`, `CLAUDE.md`, `AGENTS.md` (docs)
- `.claude/skills/` (agent skills)

## Changes

### 1. Update root `.gitignore`

Add these entries:

```gitignore
# Machine-specific (generated by spire init)
.mcp.json
.envrc
.cursor/mcp.json
.claude/settings.local.json
```

**Don't** gitignore `.cursor/rules/` or `.claude/skills/` — those are repo content, not
machine config. The `.cursor/rules/spire-messaging.mdc` file is copied from the spire repo
during init but is meant to be committed as part of the satellite's repo.

### 2. Update `.beads/.gitignore`

Add these entries:

```gitignore
# Instance-specific (generated by bd init / spire init)
metadata.json
routes.jsonl

# Credential encryption key (secret)
.beads-credential-key
```

### 3. Remove tracked files from git index

```bash
# Machine-specific editor/MCP config
git rm --cached .mcp.json
git rm --cached .envrc
git rm --cached .cursor/mcp.json
# .claude/settings.local.json — already covered by *.local in root .gitignore?
# No: *.local matches "foo.local" not "foo/settings.local.json". Add explicit entry.

# Beads instance state
git rm --cached .beads/metadata.json
git rm --cached .beads/routes.jsonl

# interactions.jsonl — this is also instance-specific (message state)
git rm --cached .beads/interactions.jsonl
```

The files remain on disk, they're just no longer tracked. After the commit, collaborators
who pull will see these files deleted from their working tree, but `spire init` will
regenerate them.

**Note on `.beads/interactions.jsonl`:** This file tracks agent message interactions. It's
runtime state, not repo content. Add it to `.beads/.gitignore` as well:

```gitignore
# Agent interactions (runtime state)
interactions.jsonl
```

### 4. Update `spire init` — ensure regeneration is robust

`spire init` already generates most of these files. Verify the following all work correctly
when the files don't exist (fresh clone scenario):

| `.mcp.json` | `installClaudeConfig()` via `patchMCPJSON()` | OK — creates if missing |

### 5. Add `spire config` subcommand

Currently missing from the CLI. Add `spire config get <key>` and `spire config set <key> <value>`
for settings that are currently implicit or flag-only.

**Where config lives:**

- `~/.config/spire/config.json` — global, per-machine (instances, shell setup)
- `.beads/config.yaml` — per-repo, committed (beads defaults)
- `.envrc` / `.env.local` — per-repo, local (secrets, identity)

`spire config` should operate on the global config by default, with `--repo` to target
the current repo's `.beads/config.yaml`.

**New config keys:**

| Key | Storage | Current state | Notes |
|-----|---------|---------------|-------|
| `identity` | global config (per-instance) | Only in `.envrc` | `spire config get identity` reads from instance config; `set` updates both config.json and .envrc |
| `dolt.port` | global config | Hardcoded 3307 in shell injection | Per-instance port for multi-hub setups |
| `daemon.interval` | global config (per-instance) | `--interval` flag on `spire up` only | Default: 2m |
| `editor.cursor` | global config | Always true | Boolean — skip cursor config in `spire init` |
| `editor.claude` | global config | Always true | Boolean — skip claude config in `spire init` |
| `mcp-server.path` | global config | Auto-detected by heuristics | Explicit fallback when detection fails |
| `dolthub.remote` | global config (per-instance) | `--dolthub` flag on `spire sync` only | Enables `spire sync` with no args |

**Implementation sketch:**

```go
// In config.go, extend SpireConfig:
type Instance struct {
    // ... existing fields ...
    DoltPort       int    `json:"dolt_port,omitempty"`       // default: 3307
    DaemonInterval string `json:"daemon_interval,omitempty"` // default: "2m"
    DolthubRemote  string `json:"dolthub_remote,omitempty"`
    Identity       string `json:"identity,omitempty"`        // default: prefix
}

type SpireConfig struct {
    Shell       ShellConfig          `json:"shell"`
    Instances   map[string]*Instance `json:"instances"`
    MCPServer   string               `json:"mcp_server_path,omitempty"`
    EditorCursor bool               `json:"editor_cursor,omitempty"` // default: true
    EditorClaude bool               `json:"editor_claude,omitempty"` // default: true
}
```

```go
// New file: cmd/spire/config_cmd.go
func cmdConfig(args []string) error {
    if len(args) < 2 {
        return fmt.Errorf("usage: spire config <get|set> <key> [value]")
    }
    switch args[0] {
    case "get":
        return configGet(args[1])
    case "set":
        if len(args) < 3 {
            return fmt.Errorf("usage: spire config set <key> <value>")
        }
        return configSet(args[1], args[2])
    case "list":
        return configList()
    default:
        return fmt.Errorf("unknown config subcommand: %q", args[0])
    }
}
```

Add to `main.go` command routing:

```go
case "config":
    return cmdConfig(args[1:])
```

### 6. Update `injectShellEnv()` to use configured port

Currently hardcodes port 3307. After this change, read from the instance config:

```go
port := inst.DoltPort
if port == 0 {
    port = 3307
}
```

This requires `injectShellEnv()` to receive the instance (or at least the port), which means
it needs to move after the instance is created in `cmdInit()`, or take the port as a parameter.

Add a comment noting that most of these are now managed by `spire config`:

```bash
# ~/.config/spire/config.json. These env vars are overrides for
# CI/automation where the spire CLI isn't available.
```

## Commit plan

Single commit:

```
chore(spi): clean up gitignore and machine-specific tracked files

- Add .mcp.json, .envrc, .cursor/mcp.json, .claude/settings.local.json
  to root .gitignore (machine-specific, generated by spire init)
- Add metadata.json, routes.jsonl, .beads-credential-key,
  interactions.jsonl to .beads/.gitignore (instance-specific)
- Remove these files from git tracking (git rm --cached)
- Files remain on disk, spire init regenerates them on fresh clones
```

## Implementation order

1. **Gitignore + git rm** — sections 1-3 (standalone, no code changes)
2. **Verify init regeneration** — section 4 (test fresh clone scenario)
3. **`spire config` subcommand** — section 5 (new feature, can be a separate PR)
4. **Port configurability** — section 6 (depends on section 5)
5. **`.env.example` update** — section 7 (trivial, bundle with anything)

Sections 1-3 are the immediate fix. Sections 5-6 are follow-up work that makes
the system more configurable but aren't blocking.
