### Subtasks

1. **K8sBackend in pkg/agent** — implements the existing Backend interface. Spawn() creates a k8s pod, Wait() watches pod completion, Kill() deletes the pod. The linchpin — everything else (store, executor, wizard, formulas) runs unchanged.

2. **Dolt StatefulSet** — PV for persistence, MySQL service on 3307, DoltHub sync via CronJob sidecar. Connection pooling for 50+ concurrent agents.

3. **ClickHouse StatefulSet** — replaces DuckDB in cluster mode. Same schema (tool_events, tool_spans, api_events, agent_runs_olap). OTel writer swap: the receiver's writeFn targets ClickHouse instead of DuckDB. Abstract behind an interface so local (DuckDB) and cluster (ClickHouse) coexist.

4. **Graph state in Dolt** — move graph_state.json from filesystem to Dolt table. Hooked-step sweep must survive pod restarts. Agent pods are ephemeral — filesystem state dies with the pod.

5. **Helm chart** — steward Deployment, dolt StatefulSet, ClickHouse StatefulSet, syncer CronJob, RBAC for agent pod creation. `helm install spire awell/spire` boots the whole thing.

6. **Secrets management** — API keys, git tokens, DoltHub creds via k8s Secrets. Per-repo token isolation. ESO (External Secrets Operator) support for Vault/AWS SM.

7. **Network policies** — agent pods → dolt (3307), agent pods → steward (4317 OTLP), steward → ClickHouse. Nothing else.

8. **Agent pod resource tiers** — implement gets 4Gi, review gets 1Gi, plan gets 512Mi. Pod priority classes for P0 beads.

9. **Bootstrap and smoke test** — helm install → file a bead → agent pod spawns → work completes → bead closes. End-to-end in a test cluster.

- Test: K8sBackend.Kill deletes the pod

- Test: ClickHouse writer produces same results as DuckDB writer for identical input
- Integration test: helm install in kind/minikube, file bead, verify agent pod runs
- Test: network policies allow required traffic, block everything else

### Exit criteria
- `helm install` boots a working Spire cluster
- Agent pods spawn, execute, and terminate correctly
- Metrics flow through ClickHouse
- Graph state survives pod restarts
- Smoke test: file bead → merged to main, entirely in-cluster
