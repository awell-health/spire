# v0.51.0

## Fixes

- **Agent spawn resolves the spire binary robustly** — child-agent spawning
  (apprentice, sage, cleric, reviewer) resolved the `spire` binary with a bare
  `os.Executable()` and no fallback. When the daemon/steward is launched from
  the macOS `Spire.app` bundle, `os.Executable()` reports the bundled
  `.../Contents/Resources/bin/spire` path; if that artifact is stale or missing,
  every dispatch died with `fork/exec ...: no such file or directory`. A new
  `agent.ResolveSpireBinary()` resolves via `SPIRE_BIN` override → `os.Executable`
  (with an on-disk existence check) → `exec.LookPath("spire")`, matching the
  resolution `spire up` already used. Applied in `pkg/agent.Spawn` and the
  wizard's review-feedback send path.
- **`reset --hard <epic>` now tears down child worktrees and branches** — the
  hard-reset sweep was keyed only on the reset target's own ID
  (`feat/<epic>`, `.worktrees/<epic>-*`), so subtask children — which carry
  their own bead IDs — kept their `feat/<child>` branches and
  `.worktrees/<child>-feature` trees. Those survivors then collided on
  `git branch -f` at the next implement wave. The reopened children are now
  swept alongside the epic.
- **Worktree/branch cleanup targets the bead's repo, not the process cwd** —
  `reset` (hard + soft) and `spire resolve` resolved `.worktrees/` and the git
  repo from `os.Getwd()`, so running the command from anywhere other than the
  repo root silently tore down the wrong tree (or nothing) while still
  reporting success. Cleanup now resolves the bead's bound repo via the
  prefix→instance registry (preferring the persisted graph-state `RepoPath` on
  the soft path) and prints the repo it operates on; it falls back to the cwd
  only when the bead's prefix is not locally registered.
