# v0.45.1

Bug-fix release. Closes P0/P1 bugs that surfaced shipping v0.45.0 to
cluster — the apprentice handoff contract, bundle tmp-file handling
in linked worktrees, helm chart gaps, and a couple of drift issues
between cluster and laptop configurations.

## Fixes

- **Apprentice transport contract aligned across wizard, executor,
  and tower defaults** (`spi-9y8eh`) — the wizard's apprentice-mode
  exit now honors `ApprenticeConfig.EffectiveTransport()`: calls
  submit (bundle) or pushes the feat branch. Dispatch-children
  tolerates both. Previously the knob was wired nowhere and cluster
  dispatches failed with "no apprentice signal for …".
- **Bundle tmp-file handling moved into `pkg/git`** (`spi-g54n4`) —
  `applyApprenticeBundle` no longer reaches into `.git` (which is a
  file, not a directory, in linked worktrees). Executor streams
  bundle bytes through a new `pkg/git` reader API; `pkg/git` owns
  tmp-file placement and cleanup.
- **K8s backend uses the Helm-generated credential secret**
  (`spi-xfp1h`) — backend references aligned with chart output
  (`<release>-credentials` + `ANTHROPIC_API_KEY_DEFAULT` /
  `GITHUB_TOKEN` keys). Pods no longer fail with
  `CreateContainerConfigError`.
- **Helm chart ships PriorityClass templates** (`spi-g6cmz`) —
  wizard pod spawn previously rejected with "no PriorityClass with
  name spire-agent-default was found".
- **Steward dispatches `status=ready` beads** (`spi-y0048`) —
  default `GetReadyWork` filter now includes `status=ready`. Fixes
  the "ready: 0 beads" silent stall in steward cycles.
- **Cluster filesystem layout matches laptop convention**
  (`spi-bx5qp`) — tower config persists across init/main containers,
  `.beads/` lives under the database subdir so `BeadsDirForTower`
  resolves.

## Improvements

- **Complete `SpireAgent` → `WizardGuild` rename** (`spi-9gy44`) —
  strips vestigial capabilities no-op; chart, kustomize, code
  comments, and docs now consistent.
