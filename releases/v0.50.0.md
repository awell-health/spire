# v0.50.0

## Features

- **Generic agent conventions in `spire doctor --fix`** — scaffolding is now
  tool-agnostic instead of Claude-specific. Work instructions go in `AGENTS.md`,
  skills install as real directories under `.agents/skills`, and tool-specific
  directories like `.claude/skills` are populated with symlinks back to the
  canonical skill. All driven by a new `scaffold` block in `spire.yaml`
  (`instructions_file`, `skills_dir`, `skills_symlinks`) — generic by default,
  with the legacy Claude-only layout available via config.

## Fixes

- **Broken skill symlinks now self-repair** — `doctor --fix` rebuilds stale or
  dangling skill symlinks in place rather than leaving them permanently broken,
  and never clobbers an existing real file or directory.
- **`doctor --fix` no longer reports the removed `spire-work` skill as missing**
  — dropped the dead required-skill check that referenced a skill deleted long
  ago and bundled nowhere, so it could never be satisfied.

## Improvements

- Documented the `scaffold` block and the legacy opt-out in `docs/spire-yaml.md`.
