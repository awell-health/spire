# v0.42.2

Patch release: operator/chart namespace-portability fix so managed-agent pods actually spawn and reach the shared dolt server when the chart is installed in any namespace other than `spire`. Without this, every helm install in a non-default namespace created wizard pods that silently failed to connect to dolt.

## Fixes

- **`operator/controllers/agent_monitor.go`: wizard pod env now namespace-portable** (`spi-21qa5`).
  - `DOLT_HOST` was hardcoded to `spire-dolt.spire.svc` in 6 places across `buildWorkloadPod` / `buildEpicPod` / `buildReviewPod`. Now derived from the operator's `--namespace` flag via `fmt.Sprintf(…, m.Namespace)`.
  - Added `BEADS_DOLT_SERVER_HOST` / `BEADS_DOLT_SERVER_PORT` / `BEADS_DIR` to wizardEnv + sidecarEnv. Without these, `bd` inside the wizard pod fell back to `127.0.0.1:0` and attempted to auto-start a local dolt server, failing on permissions.
- **`seed-beads` initContainer now works on alpine:3.20** (`spi-5pgqy`).
  - Wrapped `git init` with `|| true` — alpine:3.20 ships without `git`, exit 127 was blocking every managed-agent pod.
  - Added `chown -R 1000:1000 /data` so the wizard container (non-root UID 1000) can write to its own state dir.

End-to-end verified on minikube: bead filed on laptop → pushed to cluster dolt → operator dispatched wizard pod → wizard cloned repo, claimed bead, wrote file, pushed feature branch to GitHub.
