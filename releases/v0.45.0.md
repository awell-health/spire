# v0.45.0

Review-pipeline correctness pass plus apprentice bundle handoff. This
release tightens the sage / arbiter / wizard-claim contracts introduced
in v0.44.0 so verdicts and claims actually mean what they say, wires
the wizard to fetch apprentice bundles from BundleStore, and widens
agent observability with per-invocation Claude stream logs and richer
OLAP fields.

## Features

- **Apprentice bundle via BundleStore** (`spi-rfee2`) — wizard now
  fetches the apprentice's produced bundle from `BundleStore` and
  applies it into the staging worktree; wired into every dispatch site
  with delete-after-merge. Replaces the old implicit pickup from the
  apprentice's worktree. Wiki page added.
- **Per-invocation Claude stream logs** (`spi-9wgua`) — apprentice
  claude tool-stream is now teed to disk at a per-invocation path, so
  failures are post-mortem-able without re-running the agent.
- **Richer `agent_runs` OLAP** (`spi-e24e9`) — `turns`, `max_turns`,
  `stop_reason`, and `cache_read_tokens` / `cache_write_tokens` are now
  plumbed from the Claude result event through `ClaudeMetrics` →
  `result.json` → Dolt `agent_runs` → `agent_runs_olap`. ClickHouse
  migrations included.

## Fixes

- **Arbiter verdicts are binding** (`spi-vr65e`) — arbiter decisions now
  flow through review-round storage with `source="arbiter"`, and sage
  writers refuse to overwrite them. Before this, a post-arbiter sage
  reject could silently override the arbiter.
- **Sage accept/reject uses the review-round contract** (`spi-wtike`) —
  `spire sage accept` / `spire sage reject` now write via the same
  review-round verdict path as arbiter, instead of the legacy metadata
  shape. Reject requires `--feedback`.
- **Atomic wizard claim** (`spi-y9g3p`) — `spire wizard claim` creates
  the attempt bead and flips status in a single transaction, closing
  the race where two wizards could both claim the same task.

## Improvements

- **Retire `wizard-run` / `wizard-review` shims** (`spi-x2ou7`) — the
  legacy hyphenated verbs are gone; the same entry points live as
  hidden role-scoped subcommands under `spire wizard`. Scripts calling
  the old names will fail loudly rather than warn.
- **De-flake `TestSummonLocal_*`** — `summonLocal` now has a
  `summonSpawnFunc` test seam so unit tests don't fork/exec the test
  binary with the parent's `SPIRE_CONFIG_DIR`. Removes the "directory
  not empty" cleanup race that intermittently failed CI.
