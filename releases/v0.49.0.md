# v0.49.0

A structurally large release. The **bead lifecycle kernel** lands as the
single source of truth for status transitions: `pkg/lifecycle` introduces
typed events, predicates, an evaluator, formula `lifecycle` blocks, an
`on_complete_match` DSL with typed literals, four new statuses, and a CI
gate that blocks new direct `bead.status` writes outside the kernel. Every
production status write across executor, wizard, store, recovery, and board
flows through it; the legacy `hooked` status is gone, replaced by
predicate-driven dispatch via `lifecycle.DispatchableBeads`. **Auth pool**
ships as a new `pkg/auth/pool` foundation — selector kernel
(Pick/Release/Heartbeat), flock-based ownership, sync.Cond + inotify wake
primitive, rate-limit event parser for the Claude JSONL stream, stale-claim
sweep, steward sweep goroutine — wired into the wizard dispatch path. The
**cleric** failure-recovery loop reaches runtime: a `wait` formula
primitive, formula + prompt-builder + action handlers, promotion/demotion
learning, HITL gate resume, configurable retry cap, and loop-break on
closed recovery beads. **Bead log artifacts** become first-class: manifest
+ local/GCS store substrate, agent log capture contract, retention and
redaction policy, helm/GKE `logStore` plumbing, passive cluster log
exporter, gateway log API with live follow, board/CLI consumers. The
**workshop desktop / HITL review surface** lands its first slice: action
catalog (resummon, dismiss, update_status, comment_request, reset_hard),
matching gateway endpoints, and Phase-1 `/api/v1/workshop/*` routes. Graph
context expands inline into all five role prompts, backed by hardened
bead-close, commit, and attempt-record seams plus a new `spire graph` CLI.

## Features

- **Lifecycle kernel** (`spi-d7w4qr`, `spi-k33ims`, `spi-2r1ump`, `spi-91ohmn`, `spi-i14h5h`, `spi-u9iwt4`, `spi-kywtm2`, `spi-lkeuqy`, `spi-59fenc`, `spi-cooki3`, `spi-rbsflc`, `spi-p5u6xh`, `spi-r8m3ch`, `spi-jzs5xq`, `spi-x7c67k`) — `pkg/lifecycle` defines status taxonomy, events, errors, predicates, and an evaluator with `RecordEvent`. Formula TOML gains `[steps.X.lifecycle]` blocks, an `on_complete_match` DSL with typed literals, and a load-time warn for orphaned status targets. Four new statuses join the taxonomy. Status writes across `pkg/store`, `pkg/recovery`, `pkg/board`, `wizard.go`, `action_dispatch.go`, `executor_dag.go`, and `graph_interpreter.go` migrate to lifecycle predicates and writers; steward and gateway closed-status reads route through `lifecycle.IsMutable`; lifecycle audit logging and `ApprenticeNoChanges` event added. CI gate blocks new direct `bead.status` writes outside `pkg/lifecycle`. `hooked` status removed from production code; steward dispatch reshapes around `lifecycle.DispatchableBeads` with a legacy fallback.
- **Auth pool** (`spi-z3nui7`, `spi-dx47xa`, `spi-4z3v1n`, `spi-ntwix1`, `spi-qly1mt`, `spi-sbcnlb`, `spi-u254wr`, `spi-z5nvii`, `spi-xmvz1s`, `spi-fi90v1`) — new `pkg/auth/pool`: foundation types and flock helpers, sync.Cond + inotify wake primitive, config loader for `auth.toml` with legacy migration, selector kernel (Pick/Release/Heartbeat), heartbeat helper for in-flight claims, rate-limit event parser for the Claude JSONL stream, stale-claim sweep helper, steward sweep goroutine, `config auth` CLI verbs (probe, migrate, enhanced show), and wizard dispatch wired through the pool.
- **Cleric runtime** (`spi-h2d7yn`, `spi-egw26j`, `spi-hhkozk`, `spi-kl8x5y`, `spi-skfsia`, `spi-1u84ec`, `spi-9eopwy`) — cleric foundation: `awaiting_review` status, `wait` formula primitive, wizard hook-and-exit on step-failure, agent log capture contract (RunContext + path derivation). Cleric runtime: formula, prompt-builder, action handlers, steward dispatch. Promotion/demotion learning loop. HITL gate resume + real action execution. Pending outcomes finalize on step failure. Configurable retry cap (default 25). Loop-break on closed recovery beads.
- **Bead log artifacts** (`spi-b986in`, `spi-cmy90h`, `spi-hzeyz9`, `spi-k1cnof`, `spi-j3r694`, `spi-4jmbsc`, `spi-bkha5x`) — log artifact substrate (manifest + local/GCS store); retention, redaction, and visibility policy; helm/GKE `logStore` configuration and credential plumbing; passive cluster log exporter (with tailer offset tracking); gateway bead log API backed by artifact manifests, with live follow; board and CLI consume artifacts via the gateway.
- **Workshop desktop + action catalog** (`spi-wrjiw6`, `spi-sn0qg3`) — action catalog with gateway endpoints for `resummon`, `dismiss`, `update_status`, `comment_request`, `reset_hard`; gateway endpoints for the desktop HITL review surface, surfacing gate decision on peer recoveries for the banner heuristic; Phase-1 `/api/v1/workshop/*` endpoints; workshop desktop view design + plan + handoff bundle.
- **Graph context expansion** (`spi-rni8lr`, `spi-tyekqs`, `spi-gynkyj`, `spi-7yv2w0`) — wires inline graph-context floor into all five role prompts; hardens bead-close, commit, and attempt-record seams to support reliable graph-context expansion; adds tool-call audit + close-time commit metadata sweep; `spire graph` walks bead neighborhoods with commit-metadata expansion; parent attempts heartbeat during child waits.
- **Generalized gateway reset** (`spi-kntoe1`) — reset endpoint accepts `--to`, `--force`, `--set`, and `--hard`.
- **`spire file --caused-by` flag + validated `--design`** (`spi-9lm1i3`) — captures dependency provenance at file-time.
