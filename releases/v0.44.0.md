# v0.44.0

Role-scoped CLI taxonomy — the `spire` CLI is now organized by agent
role, and agent sessions only see their own role's command catalog.

## Features

- **Role-scoped CLI taxonomy** (`spi-o475n`) — each of the five agent
  roles gets its own cobra parent command with scoped subcommands, and
  the executor sets `SPIRE_ROLE` at spawn time so each session's hook
  surfaces only that role's catalog.
  - `spire apprentice submit [--bead <id>] [--no-changes]` — adopt the
    existing apprentice delivery verb into the role-scoped parent.
  - `spire wizard claim <bead>` — atomic attempt-bead creation +
    `status=in_progress`.
  - `spire wizard seal <bead> [--merge-commit <sha>]` — record
    `merge_commit` + `sealed_at` on the task and close the open attempt.
  - `spire sage accept <bead> [comment]` /
    `spire sage reject <bead> --feedback <text>` — record review-round
    verdicts with required feedback on reject.
  - `spire cleric diagnose <bead>` /
    `spire cleric execute --action <name>` /
    `spire cleric learn <bead>` — drive the cleric failure-recovery
    state machine.
  - `spire arbiter decide <bead> [--verdict accept|reject|custom]` —
    record a binding arbiter verdict with a `source="arbiter"` marker
    that downstream sage writers refuse to overwrite.
- **Role-aware scaffolder hooks** (`spi-o475n`) — `.claude/spire-hook.sh`
  and the Codex equivalent now read `SPIRE_ROLE` on `SubagentStart` and
  emit only the target role's command catalog. Cross-role isolation is
  enforced by tests in `pkg/scaffold/`: a wizard session never sees
  `sage accept`, an apprentice session never sees `wizard claim`, etc.
- **`pkg/scaffold` catalog package** (`spi-o475n`) — a new leaf package
  holds the per-role command catalogs as static data.
  `RenderHookInstructions(role)` emits bash-safe plaintext for the
  scaffolder; `RenderMarkdown(role)` emits docs-ready markdown for
  `docs/cli-reference.md`. One source of truth for hooks and docs.
- **Rewritten `docs/cli-reference.md`** (`spi-o475n`) — reorganized into
  role-grouped sections in the order Apprentice → Wizard → Sage →
  Cleric → Arbiter → Artificer → Common → Archmage → Deprecated.
  Signatures match `pkg/scaffold/catalog.go` exactly.

## Deprecations

The following hyphenated legacy verbs still work but now print a
deprecation warning to stderr. They will be **hard-removed in v1.0**.

- `spire wizard-merge` → use the new review flow (merge orchestration
  runs inside the wizard formula; seal with `spire wizard seal`)
- `spire wizard-epic` → use the new review flow (epic orchestration is
  driven by `spire summon` plus per-attempt `wizard claim` / `seal`)

## Upgrade notes

- Agent templates (`CLAUDE.md`, `SPIRE.md`) now point at
  `docs/cli-reference.md` for the role-grouped catalog and explain the
  `SPIRE_ROLE` env var. Existing agents that read these files will pick
  up the new taxonomy on their next session start.
- No bead data migrations. The new commands read and write existing
  metadata keys (`review_verdict`, `merge_commit`, `sealed_at`,
  `apprentice_signal_<role>`, `arbiter_verdict`, `cleric_state`).
- Hyphenated legacy verbs continue to work for this release line. Move
  off them before v1.0 ships — the warning names the replacement verb.
