# v0.42.3

## Fixes

- **Operator: AgentMonitor self-heals `CurrentWork` from running pods after restart** — previously, losing in-memory state during operator restart caused the reaper to mark running wizard pods as stale and delete them. Now reconstructs ownership from live pods (`spi-vpwex`).
- **Operator: `WorkloadAssigner.getSchedulableSet` now initializes the bead store** — crashed with `store not initialized — call Ensure() first` even when `--beads-dir` was set. Wires `store.BeadsDirResolver` into the operator and fails fast on misconfiguration (`spi-0kxhv`).
- **Chart: operator RBAC names now release-scoped** — `ClusterRole` and `ClusterRoleBinding` were hardcoded to `spire-operator`, which broke multi-tenant installs (contradicted `docs/HELM.md`). Names now derive from the release (`spi-a8x4p`).
- **`spire summon` advances `open`/`ready` beads to `in_progress`** — the status-transition switch only handled `hooked` beads, so fresh summons of `open` beads (and dismiss+resummon sequences) left status unchanged, leaving active work stuck in the backlog lane of `spire board`. Added the missing cases + a regression test (`spi-corqy`).
