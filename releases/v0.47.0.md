# v0.47.0

Cluster-attach v1 lands. Cluster-native orchestration converges on
operator-owned dispatch ‚Äî `WorkloadIntent` carries explicit
`Role`/`Phase`/`Runtime` through a single allowlist; the operator
routes by `(Role, Phase)`, not `formula_phase`; cluster control paths
fail closed when seams are missing instead of falling back to direct
spawn. The HTTP gateway is now a real surface: a `pkg/gatewayclient`
over HTTPS, complete REST endpoints (`/beads`, `/summon`, `/ready`,
`/trace`, `/lineage`, `/comments`), TLS + Ingress + ManagedCertificate
templates, and a separate deployment for the gateway workload. Two-slot
Anthropic auth is config-driven end-to-end: `spire config auth`,
summon-time slot selection, pod-builder routes the slot to the right
env var, and 429 auto-promotes to the fallback. The `DeploymentMode`
contract (`local` / `attached` / `operator`) is now explicit at every
dispatch boundary, and the wizard exclusively owns attempt-bead
lifecycle through `pkg/beadlifecycle`. A 12-section GKE install
runbook lands alongside the cluster-attach docs. `/api/v1/metrics`
exposes formula performance + lifecycle SQL; recording surface stamps
merge/finish timings, validate-design / enrich-subtasks phases, review
skips, auto-approve, waitForHuman, and Anthropic rate-limit events with
canonical `parent_run_id` linkage. Recovery converges in-wizard; debug
recovery surface lands behind a non-debug-tower guard. Legacy v2/v3
formula aliases are gone.

## Upgrade notes

- **Gateway is opt-in.** The HTTP gateway is deployed separately; enable it in your helm values if you want to use REST endpoints.
- **Two-slot auth requires configuration.** Define auth slots in `spire config auth add` before using credential selection at summon time.
- **Deployment mode is explicit.** Set `deployment_mode` in `tower create` or helm values; local is the default for backward compatibility.
- **Inline recovery is now default.** Recovery cycles execute inside wizard; opt-in to legacy cleric dispatch via formula if needed.
- **Cache PVC is canonical.** Cluster-native deployments should use PVC; repo-bootstrap is retired. Existing deployments continue to work.
