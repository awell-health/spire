# v0.47.0

The three-mode deployment contract lands. Local, cluster-native, and
attached modes are now first-class, pinned by seam tests at every
dispatch boundary (operator, steward, apprentice pod builder, runtime
contract). The wizard absorbs the cleric ‚Äî recovery runs in-wizard as
policy dispatch, the cleric formula is retired, and a new
cluster-resource-health pattern lets the operator file recovery wisps
when k8s workloads get stuck. A `spire debug recovery` surface makes
all of this testable. The `spi-hido38` epic closes the remaining
agent_runs recording gaps so DORA, per-phase timing, and parent-child
trace reconstruction have complete input.

## Features

- **Three-mode deployment contract** (`spi-4cn9c`, `spi-56c9q`,
  `spi-cvrz0`, `spi-4tfdo`, `spi-5r8ik`, `spi-v8l62`, `spi-njzmg`,
  `spi-9zkal`, `spi-197p2`, `spi-pg632`, `spi-i41xl`, `spi-xl20k`,
  `spi-07k2s`, `spi-iri62`, `spi-j9zsd`, `spi-gvrfv`) ‚Äî `DeploymentMode`
  (local / cluster-native / attached) is an explicit contract carried
  through every seam: workload intent, repo identity, steward dispatch,
  apprentice pod builder, attempt claim-then-emit. The operator owns
  an intent reconciler; the legacy scheduler is gated by mode.
  `spire tower create` and Helm write `deployment_mode` explicitly,
  cluster-only CLI lives under `spire cluster`, and the test guild
  retires repo-bootstrap in favor of `WizardGuild.Spec.Cache`. Seam
  tests pin operator/scheduler, steward-mode dispatch, pod shape, and
  runtime+transport parity.
- **Wizard absorbs cleric** (`spi-cwgiy9` epic: `spi-icgqhi`,
  `spi-tlj32a`, `spi-xezdq4`, `spi-t5gsg`) ‚Äî the cleric formula is
  retired. Recovery now runs in-wizard as policy dispatch
  (`runRecoveryCycle`, mode executes, crash-resume). Apprentice and
  cleric-worker share the same bundle handoff as wave apprentices.
  Step dispatch validates the workspace; resummon reconciles. Cleric
  decide drops git-state heuristics and routes through Claude with
  diagnosis context.
- **Cluster-resource-health recovery** (`spi-3w7d8`, `spi-2bgsm`,
  `spi-htay5`, `spi-p18tr`, `spi-w860i`, `spi-k62ay`) ‚Äî `pkg/recovery`
  grows resource-scoped recoveries (classify + diagnose). The operator
  manages a pinned-identity bead lifecycle for `WizardGuild.Cache` and
  files a recovery wisp when the refresh Job exhausts its backoff.
  End-to-end test pins: cache-refresh failure ‚Üí wisp ‚Üí cleric dispatch
  ‚Üí resolution. Pattern documented in ARCHITECTURE + VISION-CLUSTER.
- **Debug recovery surface** (`spi-yv198`, `spi-fetey`, `spi-zyz70`,
  `spi-z89if`, `spi-jtctu`, `spi-536o4`) ‚Äî new `spire debug` parent
  command with `recovery new` (author synthetic recovery beads),
  `recovery dispatch` (foreground cleric run), and `recovery trace`
  (post-run outcome viewer). Debug commands refuse to run against
  non-debug towers.
- **Reset semantics formalized** (`spi-j7juo`, `spi-tmv38n`) ‚Äî `spire
  reset` is decoupled from `summon`, `--to` rewind semantics are fixed
  and documented, and `--to --force` + `--set` land for manual operator
  override of the step-output precondition.
- **agent_runs recording gaps closed** (`spi-hido38` epic: `spi-wyktto`,
  `spi-b1357m`, `spi-e8hyw1`, `spi-mnmo6y`, `spi-obtep6`, `spi-iylbac`,
  `spi-6dzcvb`, `spi-0w3las`) ‚Äî `record.go` grows timing bucket
  options; merge/finish/retry-timing stamped in `graph_actions.go`;
  auto-approve/skip/waitForHuman + review_seconds stamped on review
  phases; validate-design/enrich-subtasks stamped in
  `executor_design.go`; Anthropic API rate-limit events recorded as
  `api_events` rows. Table-driven audit pins `ParentRunID` linkage
  across every `recordAgentRun` call site. Recording surface +
  attribution documented in METRICS.md.
- **BundleStore GCS backend** (`spi-iyykmx`, `spi-2p0ljw`) ‚Äî GCS
  backend for apprentice bundles, wired through Helm via a shared GCP
  SA credential (`gcp.*`) consolidated with `backup.gcs`.
- **Operator CRDs + RBAC from source** (`spi-2sk82u`, `spi-3shzb7`) ‚Äî
  CRDs generated from Go types via controller-gen; RBAC generated from
  kubebuilder markers. CI check-clean guards prevent drift.
- **`spire comment` CLI** (`spi-hiyt6m`) ‚Äî archmage-authored comments
  via the CLI, consistent with `spire file` / `spire update` identity.
- **Claude adapter event coverage** (`spi-eoofh`) ‚Äî hook events,
  `rate_limit_event`, and tool-result summarization now flow through
  the Claude adapter.

## Fixes

- **Cluster bootstrap hardening** (`spi-lfkfgh`, `spi-69b6ge`,
  `spi-2xf158`, `spi-ey3obm`, `spi-19v3oa`, `spi-pdrvsh`) ‚Äî `cluster bd
  init` now routes through server mode (embedded Dolt needs CGO that
  the cluster image lacks); `IsBlankDB` uses a positional SQL parser
  instead of a hardcoded column-alias allowlist; `attach-cluster
  --bootstrap-if-blank` applies the spire-schema extensions (`repos`,
  `agent_runs`, `bead_lifecycle`) that were previously missed;
  hardcoded cluster paths outside Helm helpers are eliminated from
  scripts and pod env.
- **Operator CRD + RBAC drift** (`spi-8fvhv`, `spi-p5nr8i`,
  `spi-7ctfd`, `spi-t8a57s`) ‚Äî `helm/spire/crds/wizardguild.yaml` now
  includes the cache schema (was silently blocking cluster-native
  cache bootstrap); ClusterRole grants `batch/jobs` verbs needed for
  cache-reconciler Job creation; `WizardGuild.Status.Cache.Revision`
  is populated on first successful refresh; `ClusterDispatchConfig` is
  constructed via a lazy factory + Dolt outbox so cluster-native
  dispatch actually wires up.
- **Board metrics display** (`spi-7tqx4r`, `spi-urjdiq`) ‚Äî formula
  perf SQL and the per-step timing panel render correctly; lifecycle
  P50/P95 no longer renders NULL, synthetic attempt beads are excluded
  from aggregates, and P50>P95 inversion from the min/max mixup is
  fixed.
- **Late epic inject + resummon** (`spi-g4yi6j`) ‚Äî late `spire inject`
  into a running epic now dispatches; the close-cascade guard prevents
  epic children from closing without landed code.
- **Wizard log + label cleanup** (`spi-4ytwi`, `spi-tayh7`) ‚Äî sibling
  wizard log filename and claude subdir naming now share one source of
  truth; the Claude-stdout tee is removed and per-subtask labels embed
  the bead ID.
- **`spire update` on closed beads** (`spi-3i528v`) ‚Äî `spire update`
  no longer refuses updates on closed beads; reopen works again.
- **Board comment input** (`spi-f5k2xj`) ‚Äî comment input renders as a
  multiline modal overlay instead of a single-line field.
- **Apprentice commit hygiene on epic branches** (`spi-03arfc`) ‚Äî
  commit-hygiene check is scoped to `StartSHA..HEAD` so review-fix
  apprentices on epic branches aren't blocked by prior subtask
  commits.

## Upgrade notes

- **`spire tower create` and Helm now write `deployment_mode`
  explicitly.** Existing towers continue to work; if you want to pin
  the mode, re-create or update the tower config.
- **The cleric formula is retired.** Any custom automation that
  dispatches clerics via the old formula path must switch to in-wizard
  recovery dispatch ‚Äî see `spi-icgqhi`.
- **`spire debug` commands refuse to run against non-debug towers.**
  Use a debug tower when authoring synthetic recovery beads or running
  foreground cleric dispatch.
- **`spire reset` semantics changed.** `--to` now rewinds as
  documented; `--force` and `--set` are the escape hatches for manual
  operator override.
