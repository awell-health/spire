# v0.41.1

## Fixes
- **Recovery rebase now honors the bead's base branch** — the cleric's rebase
  action walked to `origin/main` unconditionally, ignoring the `base-branch:`
  label on the bead (or its parents). Renamed `actionRebaseOntoMain` →
  `actionRebaseOntoBase`, plumbed a `BaseBranch` field through
  `RecoveryActionCtx`, and resolved the base by walking the parent chain
  before falling back to `repoconfig.DefaultBranchBase`. (`spi-rswvi`)
- **Wizard no longer panics on cleanup after merge-race exhaustion** —
  `HookStepBead` / `UnhookStepBead` were not wired into
  `buildExecutorDeps`, so the graph interpreter hit a nil-func panic in
  deferred cleanup. Wired both; added a reflection-based regression test in
  `executor_bridge_test.go` that fails if any `executor.Deps` field is left
  nil. (`spi-qdmy8`)

## Improvements
- Replaced remaining hardcoded `"main"` fallbacks across the steward
  (`DetectMergeReady`, merge orchestration), wizard (`ResolveRepo`),
  operator `agent_monitor`, `tower`, `register_repo`, and `summon` with
  `repoconfig.DefaultBranchBase` / `repoconfig.ResolveBranchBase` so the
  canonical source of truth lives in one place.
