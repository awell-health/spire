## Features

- **Steward auto-summons clerics for failed beads** — when the steward detects hooked beads with failure evidence, it automatically summons a cleric to investigate and recover (`spi-1xygz`)
- **Cleric resumes wizard staging worktree** — clerics now resume the original wizard's staging worktree instead of creating a separate `recovery/<beadID>` branch, preserving work context (`spi-17xbl`)
- **Cleric retry loop** — cleric formula includes a retry loop gated on `completed_count < 4`, allowing multiple recovery attempts before giving up (`spi-o5ewn`)
- **`spire update --parent` flag** — reparent beads after creation with `spire update <id> --parent <parent-id>` (`spi-pvum7`)
