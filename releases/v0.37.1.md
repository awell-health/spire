# v0.37.1

> Metrics pipeline fixes, dual-signal OTel, prefix inheritance, interrupted bead visibility.

## Features

- **Dual-signal OTel receiver** — accepts both OTLP logs and traces; Claude sends tool_result via logs (stable), trace spans via traces (beta). Both channels write to DuckDB (`spi-g1q3y`)
- **Per-step trace metrics** — `spire trace` shows duration, cost, tokens, and tool calls per step (`spi-kowpq`)
- **Review metadata** — verdicts and findings stored as structured metadata, not description text parsing (`spi-w00yn`)
- **Improved failure classification** — 8 new specific failure patterns replace generic "unknown" (`spi-hj1kt`)
- **Canonical metric set** — documented in docs/METRICS.md, the contract for CLI, board, and future MCP/dashboard (`spi-hqc6e`)
- **CLI/board metrics reconciled** — both read from DuckDB, no more divergent numbers (`spi-bdyyk`)

## Fixes

- **Child beads inherit parent prefix** — subtasks of oo- epics no longer get spi- prefix, fixing cross-repo wizard failures (`spi-fh38d`)
- **Interrupted beads visible on default board** — no longer hidden behind ViewLower tab switch (`spi-ywc65`)
- **Duplicate step beads on resummon** — reuses existing step-type children instead of creating new ones
- **DuckDB lock conflict resolved** — single writer service with open/close per write and retry (`spi-y0kdr`)
- **OTel exporter env vars** — Claude Code now gets OTEL_TRACES_EXPORTER, OTEL_LOGS_EXPORTER, protocol=grpc
- **DORA metrics fixed** — deploy frequency, failure rate, lead time now reflect actual merge activity (`spi-wob6d`)
- **Formula Performance populated** — daily_formula_stats query and TIMESTAMP handling fixed (`spi-wob6d`)
- **ETL data quality** — repo derived from bead_id, formula_name normalized (`spi-gtmud`)
- **OTel log pipeline wiring** — log events correctly parsed and written to tool_events (`spi-343as`)
- **anyValueToString zero-value bug** — OTel attributes with 0/false values no longer silently dropped (`spi-g1q3y`)
