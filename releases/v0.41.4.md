# v0.41.4

## Features
- **Pluggable OLAP backends (DuckDB local, ClickHouse cluster)** — extracted the OLAP layer into a Writer/TraceReader/MetricsReader interface split with a factory (`spi-8w3s6`), moved DuckDB into `pkg/olap/duckdb` behind a cgo build tag (`spi-czum9`), added a ClickHouse subpackage with trace reader + integration CI (`spi-v0kn3`), and refactored consumers (`cmd/spire`, steward daemon) to go through the factory so `CGO_ENABLED=0` builds are green (`spi-4pa1g`).

## Fixes
- **Review-fix step no longer silently approves unchanged code** — when the fix apprentice's claude subprocess was killed (signal/timeout) and staged zero changes, the fix step declared success, the graph's `resets` fired, and sage re-ran against unchanged code — often flipping a round-1 `request_changes` into a round-2 `approve`. The fix now returns an error on "apprentice killed + zero staged changes," so the review DAG escalates to arbiter at max rounds instead of looping on identical code. (`spi-fljzd`)
- **`pkg/olap/db.go` now honors `//go:build cgo`** — gates the go-duckdb import behind the tag so `CGO_ENABLED=0 go build ./...` succeeds. (`spi-0o7bn`)
