- **Multi-mode TUI** ‚Äî RootModel with tab routing, BoardMode refactor with owned connection, AgentsMode with registry fetcher, MetricsMode with DuckDB, Mode interface, store.Open factory (`spi-t7f3c` epic: `spi-i6gq7`, `spi-67yhs`, `spi-6v20l`, `spi-xhtqr`, `spi-i7cs4`, `spi-59jbs`)

## Fixes

- **Pipeline display from formula topology** ‚Äî replaced hardcoded `v3StepOrder` map (ZFC violation) with topological sort of formula's `needs` edges (`spi-wugyb`)
- **Tab key ownership** ‚Äî RootModel owns Tab for mode switching, modes use `1-4` for internal navigation, footer hints corrected (`spi-87a62`)
- **ETL cursor correctness** ‚Äî switched from random `id` to monotonic `started_at`, no more skipped rows (`spi-mc4nw`)
- **Tower switching propagation** ‚Äî BEADS_DIR, store connection, and metrics DB all update when switching towers (`spi-l154y`)
- **OTel write safety** ‚Äî WriteBatch acquires olap.DB mutex, prevents DuckDB write races (`spi-6irug`)
- **Materialized view retention** ‚Äî extended from 7-28 days to 90 days to match query horizons, wired tool_events to board (`spi-4uwsx`)
- **Triage repo path resolution** ‚Äî feat-branch fallback resolves repo from bead prefix, not CWD (`spi-nb78p`)
- **Agent action menu** ‚Äî Grok/Trace actions wired in parsePendingAction and executeBoardAction (`spi-mny9j`)
- **Board view actions** ‚Äî footer hints and key handlers for agents mode and blocked/interrupted views (`spi-t0m7u`)
- **Recovery board visibility** ‚Äî non-closed recovery beads shown on board (`spi-drkdr`)
- **Triage --no-review** ‚Äî triage agent is one-shot fix only, no review handoff (`spi-s0z3u`)
- **Resolve without needs-human** ‚Äî `spire resolve` works on any non-closed bead
- **parseTime=true** ‚Äî MySQL DSNs for OLAP ETL now parse datetime correctly
