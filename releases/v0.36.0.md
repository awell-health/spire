# v0.36.0

> Internal bead types, recovery triage, metrics tracking, board TUI improvements.

## Features

- **Internal bead types** — messages, steps, attempts, and reviews are now proper types instead of label-identified beads. Single `IsWorkBead()` filter replaces 5 duplicated label checks. Migration runs automatically in `spire up` (`spi-awlp5`, `spi-zp8ow`, `spi-hhcbi`, `spi-mgtlx`, `spi-uvik2`, `spi-yp73u`, `spi-c6c62`)
- **Recovery triage action** — recovery agent can dispatch a fix agent into the failing worktree to diagnose and repair, instead of resetting or escalating (`spi-yysrl`)
- **Recovery reset-hard action** — full destructive reset (worktree, graph state, branches, step beads) now available as a recovery action (`spi-fnob5`)
- **Metrics: failures, timing, tokens, tool calls** — agent_runs now tracks tool call counts (read/edit/write), failure classification, and attempt number per step (`spi-btz6q`)
- **Board tab sidebar** — Tab key toggles between alert view, normal board, and blocked/interrupted view (`spi-77zxt`)
- **Terminal pane overlay** — trace viewer opens in an overlay pane within the board TUI (`spi-r83hy`)
- **Rebase conflict resolution in merge** — `git.merge_to_main` now resolves rebase conflicts via Claude instead of escalating immediately (`spi-85sbn`)

## Fixes

- **Board filters unmigrated beads** — `skipBead()` restores label-based fallback for beads not yet migrated to internal types, plus filters recovery beads
- **Summon rejects closed/deferred beads** — positional args validated against status before dispatching (`spi-u5xmu`)
- **Steward health check skips internals** — no more SHUTDOWN attempts against step beads, attempts, and messages
- **No hardcoded max-turns** — conflict resolution and doc review no longer impose a turn budget; formula `with.conflict_max_turns` is the only knob

## Improvements

- **Helm design document** — production cluster architecture captured (`spi-8n9km`)
