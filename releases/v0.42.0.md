# v0.42.0

Subsumes the never-tagged v0.41.4 (CI didn't tag — content rolled into this release).

## Features

- **Helm chart install path matures** — chart now installs cleanly against a real DoltHub tower (`spi-0l2zr`); supports the blank-tower Use Case 2b via `spire tower create` (`spi-8x60z`); auto-provisions the remotesapi SQL user (`spi-50788`) and additional dolt users via `dolt.additionalUsers` (`spi-4ulrm`); ships a chart-managed DoltHub JWK Secret for the dolt pod (`spi-o4ne6`); Phase 1 backups via a GCS CronJob driving `dolt backup sync` (`spi-ra5em`); multi-tenant smoke test + install docs (`spi-62sda`).
- **Steward/agent/builder dockers on ghcr.io with auto-bumping appVersion** — goreleaser publishes the three images to ghcr.io (`spi-oqpqu`); chart appVersion auto-bumps on release with default image tags pinned to AppVersion (`spi-wps13`).
- **Cluster dolt remotesapi as the sync spine (pivot away from DoltHub)** — Phase 1 enables remotesapi on cluster dolt; gateway/daemon split with operator/syncer beads-dir wiring (`spi-o3uja`).
- **Tower attach to cluster remotesapi from laptop** — extracted `pkg/tower` and added `spire tower attach-cluster` (`spi-6j10j`); `spire tower attach` now accepts cluster URLs of the form `dolt://...:50051/db` and `http://...:50051/db` (`spi-ibq3g`).
- **Pluggable OLAP backends (DuckDB local, ClickHouse cluster)** — extracted Writer/TraceReader/MetricsReader split with a factory (`spi-8w3s6`); DuckDB moved into `pkg/olap/duckdb` behind a cgo build tag (`spi-czum9`); ClickHouse subpackage with trace reader + integration CI (`spi-v0kn3`); consumers refactored so `CGO_ENABLED=0` builds are green (`spi-4pa1g`).
- **Cleric recovery: retry budget, decide fixes, agentic conflict resolution, promotion policy** — `execute` now retries on error with budget=2 before `needs_human` (`spi-676a4`); fixed three interacting bugs causing decide to pin on a failing action across retries (`spi-uh5oo`); Claude-driven conflict resolver with bead/commit context and commit freedom (`spi-nghqn`); cleric promotion policy (`spi-4o4bi`).
- **Sidecar skips LLM calls when idle, defaults to haiku-4-5** (`spi-fgmps`).
- **Per-round log files for looping graph steps** — round-suffixed logs preserve history across review/fix loops instead of overwriting (`visibility`).
- **Board: `y` always yanks the bead ID; approve moves to `a → y`** — context-sensitive action menu now hosts approve entries by label (`spi-0dfl4`).

## Fixes

- **Review-fix step no longer silently approves unchanged code** — when the fix apprentice's claude subprocess was killed (signal/timeout) and staged zero changes, the fix step declared success, the graph's `resets` fired, and sage re-ran against unchanged code — often flipping a round-1 `request_changes` into a round-2 `approve`. Now returns an error on "apprentice killed + zero staged changes" so the review DAG escalates to arbiter at max rounds (`spi-fljzd`).
- **`pkg/olap/db.go` honors `//go:build cgo`** so `CGO_ENABLED=0 go build ./...` succeeds (`spi-0o7bn`).
- **Image/secret/operator wiring** — bd v1.0.2 in images; apiKey written before oauthToken in the chart secret; skip `kill()` when owner unknown (`spi-mpo8e`); `SPIRE_AGENT_IMAGE` wired into steward (`spi-0l2zr`); daemon TOCTOU + tests (`spi-ibq3g`); `isK8sAvailable()` timeout + test seam (`spi-cli5v`).
- **Scaffolder hook output emits `hookEventName`** — newly-scaffolded `.claude/spire-hook.sh` shipped a JSON envelope missing the field (`spi-19d9h`).
