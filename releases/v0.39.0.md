# v0.39.0

> Operational state model, cleric recovery agent, multi-local steward safety, and production infrastructure.

## Features

- **Operational state model** ‚Äî step beads carry status (`hooked`, `in_progress`, `closed`) instead of labels. The board, steward, and CLI all route by bead status. The `Interrupted` column becomes `Hooked`. (`spi-lqsdo`)
- **Cleric recovery agent** ‚Äî git-aware, worktree-capable recovery with cooperative wizard handoff. Runs `cleric-default` formula: collect context ‚Üí decide ‚Üí execute ‚Üí verify (polling loop) ‚Üí learn ‚Üí finish. Renamed from "recovery agent" to match RPG naming. (`spi-07z3w`, `spi-l2mp1`)
- **Multi-local steward safety** ‚Äî attempt-based execution leases with stable instance identity. Multiple stewards on different machines can scan the same tower safely. (`spi-8wc0w`)
- **Steward production modules** ‚Äî per-tower concurrency limiter, merge queue with real git operations, trust gradient per repo, A/B formula routing, health endpoints (`/readyz`, `/livez`, `/health/detailed`), cycle stats (`spi-m0ic5`)
- **K8s cluster foundation** ‚Äî Helm chart with steward Deployment, Dolt StatefulSet, ClickHouse OLAP, sync CronJob, proper bootstrap and RBAC (`spi-2jlhe`)
- **`spire review`** ‚Äî read-only context assembly from bead commit metadata: files changed, commit log, review history (`spi-ck7jh`)
- **`spire ready`** ‚Äî mark beads for steward pickup; `GetReadyWork` filters by status=ready (`spi-lccod`)
- **`spire file --design`** ‚Äî alias for `--ref`, creates `discovered-from` dep to design bead. CLAUDE.md updated to prefer `spire file` over `bd create`
- **`spire update`** ‚Äî wraps `bd update` with spire conventions (`spi-wgs80`)
- **Apprentice commit metadata** ‚Äî writes `metadata.commits` on bead at commit time for `spire review` (`spi-cids2`)
- **Board UX** ‚Äî Shift-J/K fast scroll, status-based action menus, hooked step details in inspector, pipeline icons for hooked beads, compact attention line, Backlog column (`spi-2wdfm`, `spi-4p9nl`, `spi-lccod`)
- **Graph state persistence in Dolt** ‚Äî `DoltGraphStateStore` for cluster mode (`spi-rxv2d`)
- **V2 dead code removal** ‚Äî wizard_epic files, dual v2/v3 paths, dead types all removed (`spi-b0ejk`)
- **CloseMoleculeStep removal** ‚Äî vestigial molecule tracking calls removed (`spi-vm91e`)

## Fixes

- **Resummon preserves graph state** ‚Äî no longer deletes state on resummon, wizard resumes where it left off (`spi-63aq5`)
- **Escalation preserves graph state** ‚Äî terminal escalation steps don't remove state file, enabling `reset --to` (`spi-njxph`)
- **Approve safety** ‚Äî only clears `human.approve` and `design-check` gates, not arbitrary hooked steps; correctly updates graph state CompletedCount
- **Design bead approve** ‚Äî `spire approve` on a design bead closes it; `spire summon` rejects design beads with guidance
- **Diagnose accepts hooked beads** ‚Äî requires failure evidence (recovery/alert beads), recognizes both `recovery-for` and `caused-by` link models
- **Recovery protocol fixes** ‚Äî result labels preserved for polling, graph step names mapped to wizard phases, worktree branch cleaned up on retry (`spi-10bou`, `spi-a0spf`, `spi-h6xe1`)
- **Recovery verify wiring** ‚Äî formula routes through cooperative `recovery.verify` loop instead of legacy `verify-clean` (`spi-ilz58`)
- **Helm chart bootstrap** ‚Äî steward PVC/init, Dolt repo init, ClickHouse DB creation, sync CronJob RBAC (`spi-2jlhe.1-6`)
- **Concurrency limiter** ‚Äî filters by tower via `agent.Info.Tower` field, not global count (`spi-m0ic5.4`)
- **Test isolation** ‚Äî `cmd/spire` tests never hit live dolt server; integration tests use `restoreDoltPort(t)` (`spi-vaal2`)
