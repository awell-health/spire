# v0.35.0

> Hooked recovery flow, deferred status, multi-provider formulas, canonical formula names, V2 removal complete.

## Features

- **Hooked step status** ‚Äî failed steps now park as `hooked` instead of crashing out, enabling the resolve -> steward -> re-summon recovery flow (`spi-hc82e`, `spi-1wvwe`)
- **`spire resolve`** ‚Äî new command for human gate clearance: records recovery learning, unblocks source bead, steward re-summons automatically (`spi-iwesh`, `spi-0sf6y`)
- **`human.approve` action** ‚Äî formula step gate that parks until human approves, distinct from resolve (approval vs recovery) (`spi-jzct4`)
- **Deferred bead status** ‚Äî beads can be deferred to hold them back from agents; shown dimmed at bottom of Ready column with `d` key toggle (`spi-3o6ru`, `spi-v6i7n`)
- **Per-step provider override** ‚Äî formula steps can declare `provider = "codex"` to run on different AI backends (`spi-uagp9`)
- **Inline prompt in formulas** ‚Äî `with.prompt` on `wizard.run` steps injects custom prompts wrapped with system context, enabling extensible formulas without Go code changes (`spi-f4zqn`)
- **`with` parameter interpolation** ‚Äî formula `with` fields now resolve `{steps.X.outputs.Y}` and `{vars.Z}` at runtime, enabling step-to-step data flow (`spi-7zyxu`)
- **Steward hooked step sweep** ‚Äî periodic check finds parked graphs and re-summons when the hook condition clears (`spi-uqeco`)
- **`spire summon` accepts bead IDs** ‚Äî `spire summon spi-xxx spi-yyy` works alongside the existing `spire summon N` syntax (`spi-4iz3b`)
- **`spire sql`** ‚Äî direct SQL queries against the dolt server (`spi-aa0ad`)
- **Apprentice build gate** ‚Äî apprentices must pass the build before exiting; auto-fix loop on failure (`spi-8l23h`)
- **Recovery decide agent improvements** ‚Äî wizard log context, expected outcome tracking, relapse detection (`spi-4mb51`)
- **Design-linked auto-create** ‚Äî `check.design-linked` can auto-create a design bead when none exists (`spi-e1trw`)
- **Canonical formula names** ‚Äî embedded formulas renamed: `task-default`, `bug-default`, `epic-default`, `chore-default`, `recovery-default`, `subgraph-review`, `subgraph-implement`. Legacy names resolve via backward-compat map (`spi-7f3ur`)
- **Scheduling policy dedup** ‚Äî shared `store.GetSchedulableWork` used by both steward and board (`spi-s3lx3`)

## Fixes

- **Epic children filtered from board and steward** ‚Äî only top-level beads are assignable; subtasks managed by their parent epic's formula (`spi-2y76v`)
- **Recovery learning path fixed** ‚Äî recovery beads stay open on escalate so `spire resolve` can write learnings; SQL table now queried alongside bead metadata (`spi-r414z`, `spi-1g5rl`)
- **Tower-scoped hooked step sweep** ‚Äî no more cross-tower interference on multi-tower machines (`spi-04z9c`)
- **Archmage identity** ‚Äî `promptArchmageIdentity` reads global git config, not repo-local (`spi-t3jsg`)
- **Sidecar re-parent** ‚Äî removes existing parent-child dep before adding new one, preventing multi-parent ambiguity (`spi-e0gon`)
- **`spire pull` conflict propagation** ‚Äî unresolved conflicts now correctly reported as failures (`spi-5tw3h`)
- **Parked graph step bead cleanup** ‚Äî hooked graphs no longer close step beads prematurely (`spi-ti8ky`)
- **Resummon warns about missing learning** ‚Äî nudges human to use `spire resolve` instead (`spi-qqet2`)

## Improvements

- **V2 removal complete** ‚Äî all FormulaV2 types, v2 executor code, v2 tests, and v2 embedded formulas deleted (`spi-2u66k`, `spi-itl52`, `spi-xo0jg`)
- **Steward sidecar store API migration** ‚Äî subprocess calls replaced with direct store API (`spi-b26ec`, `spi-oyr97`, `spi-u03wj`)
- **Shared process helper** ‚Äî `pkg/process` for background spawning (`spi-8el`)
- **Registry lifecycle consolidation** (`spi-1y4`)
