# v0.46.0

Observability, provider neutrality, and the cluster-side wizard pod
converge. This release lands a provider-neutral raw-transcript surface
(Claude + Codex share one logstream contract), bead-level lifecycle
analytics you can actually query, the full k8s guild-cache pipeline for
shared-workspace wizard spawns, and a cleric/recovery contract that now
fails closed instead of dropping outcomes on the floor. A layered
observability regression suite + a canonical runtime-contract audit keep
all of the above honest.

## Features

- **Provider-neutral raw transcript surface** (`spi-gbd0y` epic:
  `spi-7amue`, `spi-z70v2`, `spi-7mgv9`, `spi-molx7`, `spi-0jvyv`,
  `spi-i652t`, `spi-0ecx9`) — board inspector and `spire logs` now read
  transcripts from every provider (Claude, Codex, …) through one
  `pkg/board/logstream` adapter contract. Per-invocation transcripts
  land at `<agentResultDir>/<provider>/<label>-<ts>.jsonl` with a
  matching `.stderr` sidecar, plus a pretty reader and an end-to-end
  Codex+Claude parity fixture test.
- **Bead lifecycle analytics** (`spi-hmdwm`, `spi-ouuta`) — canonical
  filed / ready / started / closed timestamps are stamped at the store
  boundary for every bead type and synced into OLAP. `spire metrics
  --bead <id>` now breaks out filed-to-closed, ready-to-closed, started-
  to-closed, plus sage reviews / fix loops / arbiter rounds. Queue delay
  stays visible separately so API/execution slowness is distinguishable
  from scheduling delay. `spire up` runs the migration and backfills
  historical beads automatically.
- **K8s guild cache + shared-workspace wizard pods** (`spi-tpzcq`,
  `spi-bsngj`, `spi-myzn5`, `spi-jetfb`, `spi-s2dzk`, `spi-2tu4d`,
  `spi-zpnyu`, `spi-yzmq0`, `spi-wllmv`, `spi-02dbw`, `spi-xctof`) —
  `WizardGuild` CRD now owns a `CacheSpec`; the operator reconciles it
  into a PVC + refresh `Job`, and wizard pods bootstrap their workspace
  from the cache at startup. Shared-workspace spawns get a per-wizard
  PVC (`spi-zpnyu`) and the refresh/bootstrap markers converge on a
  single contract. Default is off — opt in via the CR
  (`SPIRE_K8S_SHARED_WORKSPACE`, `spi-cslm8`).
- **Canonical runtime contract + parity audit** (`spi-b9tu3`,
  `spi-ypoqx`, `spi-smk2a`, `spi-fjt2t`, `spi-wqax9`, `spi-q2kzk`,
  `spi-fopwn`, `spi-6n26g`, `spi-3ca64`, `spi-kbqme`, `spi-9wo3a`,
  `spi-sw2zd`) — introduces the `RepoIdentity` / `RunContext` types,
  routes every dispatch site through them, shares one pod builder
  between operator and pkg/agent, and guards the contract with an audit
  test + parity test lane. Adds `CLI migration guide` for explicit
  tower/prefix.
- **RunContext observability** (`spi-zm3b1`) — `bead_id`, `attempt_id`,
  `run_id`, `tower`, `prefix`, `role`, `formula_step`, `backend`, and
  `workspace_*` now thread through executor, wizard, agent, operator
  controller, and metrics logs + OTEL resource attributes. Parity
  integration test pins the matrix. Pair this with the canonical OTEL
  receiver fix (`spi-ecnfe`) to get every signal correlatable back to a
  bead.
- **Cleric repair loop contract** (`spi-9ql7l`, `spi-ka6bh`, `spi-nwfn1`,
  `spi-1rild`, `spi-gnqpf`, `spi-dz07b`, `spi-tmqf6`, `spi-usco5`) —
  splits recovery into domain (`RepairMode` / `RepairPlan` /
  `VerifyPlan` / `RecoveryOutcome`) vs policy. Cleric verify now runs
  from `RepairPlan.VerifyPlan`, decide policy lifted into
  `pkg/recovery`, and promoted recipes execute through the un-promoted
  primitives — fewer bespoke code paths per recipe.
- **Machine-readable `spire focus`** (`spi-py6k1`) — `spire focus
  --json` emits a stable shape for external tooling.
- **Explicit handoff mode** (`spi-yif4t`) — `HandoffMode` is threaded
  through every dispatch and recorded in apprentice signal payloads.
  Legacy push transport is quarantined behind the mode, not inferred.

## Fixes

- **Codex CLI invocation** (`spi-z70v2`) — `CodexProvider` now invokes
  `codex exec --json <prompt>` and parses the JSONL stream, replacing
  the stale `codex -q` call that silently stopped working upstream.
- **Canonical OTEL resource attrs** (`spi-ecnfe`) — receiver and log
  parser now accept the canonical `bead_id` / `formula_step` / `tower`
  keys emitted post-`spi-xplwy`, with legacy fallbacks explicitly
  enumerated. Previously the emitter rename left the receiver looking
  up the old keys, silently dropping correlations.
- **RecoveryOutcome writes fail closed** (`spi-nqf7z`, `spi-0nkot`,
  `spi-6wiz9`) — recovery outcomes are written through a single typed
  accessor; steward closes escalated recovery beads instead of
  re-summoning clerics in a loop; `RepairModeWorker` goes through the
  canonical runtime contract.
- **`bead_lifecycle` ETL cursor** (`spi-ouuta`) — ETL uses a composite
  `(updated_at, bead_id)` cursor so repeated syncs advance correctly
  across same-second updates.
- **Operator-managed wizard pod repo checkout** (`spi-fopwn`,
  `spi-3h1tp`, `spi-vrzhf`, `spi-x7fus`) — cluster dolt env threads into
  wizard init with the correct port default; the wizard pod WorkingDir
  points at the cloned repo; backend resolution threads the bead's repo
  path through `ResolveBackendForRepo` with a cwd-above-clone assertion.

## Improvements

- **Layered observability regression suite** (`spi-9h5rt`) — one
  `make test-observability` target runs pkg/olap contract tests,
  pkg/otel round-trip tests, and CLI reader tests (via injected typed
  readers — no SQL above `pkg/olap`). Canonical vocabulary pinned in
  `pkg/observability/obsvocab`. Missing-bead and legacy-attr paths are
  asserted to fail loudly rather than silently.
- **Wizard timeout raised to 60m** — apprentice hard timeout now matches
  the 150-turn default; was cutting long implement phases short.
- **Docs pass** (`spi-sw2zd`, `spi-xctof`, `spi-02dbw`, `spi-usco5`,
  `spi-smk2a`) — ARCHITECTURE / LOCAL / package READMEs sweep remaining
  Model A references and land the current-state wizard pod + runtime
  contract docs.

## Internal

- `pkg/recovery` now owns policy (`spi-nwfn1`); `pkg/executor` is purely
  orchestration.
- Shared pod builder between operator controller and `pkg/agent`
  (`spi-fjt2t`, `spi-wqax9`).
- Removed `agent-entrypoint.sh` and image references (`spi-d4ku6`).

## Upgrade notes

- **Restart `spire up` after upgrading.** The `bead_lifecycle` table is
  created on `spire up`; existing long-running daemons started before
  v0.46.0 will log `Error 1146: table not found: bead_lifecycle` until
  the migration runs. The errors are non-fatal (stamping becomes a
  no-op) but `spire metrics --bead` will be empty until you run it.
- **`SPIRE_K8S_SHARED_WORKSPACE` defaults off.** Opt in explicitly via
  the `WizardGuild` CR; see `spi-cslm8` if you were relying on the
  previous default.
- **Legacy push handoff is quarantined.** Callers that relied on
  implicit push dispatch must now set `HandoffMode` explicitly.
