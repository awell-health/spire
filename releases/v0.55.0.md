# v0.55.0

## Improvements

- **Agent docs no longer misstate the chore lifecycle** — since the
  introduction of the dedicated `chore-default` formula (spi-7f3ur,
  shipped in v0.35.0), chores run research → implement → review → document →
  merge, where the document step writes a page under `docs/wiki/`. The
  SPIRE.md template that `spire doctor` provisions into registered
  repos, `docs/ARCHITECTURE.md`, and `docs/spire-yaml.md` all still
  documented `chore → task-default` (implement → review → merge), so
  agents and operators were told the wrong pipeline and the
  documentation step came as a surprise. All three now document the
  actual mapping.

## Notes

- Repos provisioned before this release carry the stale SPIRE.md.
  Re-run `spire doctor` in each registered repo to refresh it.
