# v0.41.2

## Features
- **Claude subprocess log visibility** — each `claude` invocation now tees stdout+stderr to a per-invocation log file at `<agentResultDir>/claude/<label>-<timestamp>.log`, and the wizard log prints the path so you can drill from high-level orchestration down to raw model output.
- **`spire logs --claude`** — new flag lists and tails claude subprocess logs by label; pair with `--claude-file <path>` to tail a specific log.
- **Board inspector log cycling** — in `spire board`, select a bead, switch to the Logs tab, then use `h`/`l` (or ←/→) to cycle between the wizard log and every claude log for that bead (including apprentice and sage runs). Long lines now wrap instead of being truncated.
- **Markdown-rendered logs in the inspector** — log content runs through glamour, so prompts written with markdown (headers, bullets, code fences) render formatted instead of appearing as a wall of backslash-n. Logs now use the full inspector width rather than the 100-col prose cap. `claude_runner` writes each arg on its own line so prompt newlines survive into the rendered output.

## Fixes
- **Invalid model alias in config** — `claude-opus-latest` is not a real Anthropic alias; replaced with `claude-opus-4-7` in `spire.yaml` and the sage-review / fix / arbiter steps of `subgraph-review.formula.toml`.
- **Dead `WIZARD_MODEL` env var in operator** — `buildEpicPod` and `buildReviewPod` both injected `WIZARD_MODEL` into wizard pods, but it was read nowhere in the codebase — a duplicate config assertion made by the operator. Deleted both; clusters now inherit model choice from `spire.yaml` on the mounted repo.
- **Arbiter model fallback drift** — bumped the hardcoded arbiter fallback in `graph_actions.go` from `claude-opus-4-6` to `claude-opus-4-7` to match the formula. Follow-up tracked: route model choice through config rather than hardcoded fallbacks scattered across the codebase.

## Improvements
- `.gitignore` now covers `.claude/scheduled_tasks.lock` (agent runtime state) and stray `mysql:/` directories created by the mysql CLI when connecting to the local dolt server on port 3307.
